Что мы будем создавать?

Почему именно кликер?

Кликер — идеальная первая игра для изучения Godot. Она проста в реализации, но охватывает ключевые концепции: работа с UI, логика игры, сохранение данных и визуальная обратная связь.

Создаём главную сцену

Нажмите "Создать сцену" и добавьте следующие узлы:

Зачем CanvasLayer?

CanvasLayer позволяет отделить интерфейс от игровых объектов. Это гарантирует, что UI всегда будет поверх других элементов, независимо от их позиции на сцене.

Шаг 3: Написание основного скрипта

Создаём скрипт для игры

Создайте новый скрипт и прикрепите его к главному узлу (Node2D). Назовём его "clicker_game.gd".


Основные переменные 
extends Node2D

# Основные переменные игры
var coins = 0
var coins_per_click = 1
var coins_per_second = 0

# Ссылки на узлы интерфейса
@onready var coin_label = $CanvasLayer/CoinLabel
@onready var click_button = $CanvasLayer/ClickButton
@onready var per_second_label = $CanvasLayer/PerSecondLabel

func _ready():
    # Обновляем интерфейс при запуске
    update_ui()

    # Создаём таймер для авто-кликов
    var timer = Timer.new()
    add_child(timer)
    timer.wait_time = 1.0  # Каждую секунду
    timer.timeout.connect(_on_timer_timeout)
    timer.start()

func _on_timer_timeout():
    # Автоматическое добавление монет
    coins += coins_per_second
    update_ui()

func update_ui():
    # Обновляем текст счётчика
    coin_label.text = str(coins)
    per_second_label.text = "В секунду: " + str(coins_per_second)

Добавляем обработку кликов

Теперь сделаем так, чтобы при нажатии на кнопку добавлялись монеты:

Функция клика 
# Эта функция будет вызываться при нажатии на кнопку
func _on_click_button_pressed():
    coins += coins_per_click
    update_ui()

    # Добавляем визуальный эффект
    create_click_effect()

func create_click_effect():
    # Создаём эффект "всплывающего текста"
    var effect = Label.new()
    effect.text = "+" + str(coins_per_click)
    effect.add_theme_font_size_override("font_size", 48)
    effect.add_theme_color_override("font_color", Color.GOLD)
    $CanvasLayer.add_child(effect)

    # Позиционируем эффект рядом с кнопкой
    effect.position = click_button.position + Vector2(150, -50)

    # Анимация движения вверх и исчезновения
    var tween = create_tween()
    tween.tween_property(effect, "position",
        effect.position + Vector2(0, -100), 0.5)
    tween.parallel().tween_property(effect, "modulate:a",
        0.0, 0.5)

    # Удаляем эффект после анимации
    tween.tween_callback(effect.queue_free)

Важно!

Не забудьте подключить сигнал pressed от кнопки к функции _on_click_button_pressed в инспекторе Godot. Для этого перетащите кнопку в поле "Node" сигнала и выберите соответствующую функцию.

Как работает наш код?

Давайте разберём ключевые моменты скрипта:

@onready переменные — переменные, которые подключаются к узлам после их загрузки
функция _ready() — функция, которая выполняется один раз при запуске сцены
Timer — добавляет монеты автоматически каждую секунду
Tween — создаёт плавные анимации для визуальных эффектов
Сигналы — соединяют действия игрока (клик) с кодом

Профессиональный совет

Используйте @export перед переменными, чтобы они отображались в инспекторе Godot. Это позволяет настраивать значения прямо в редакторе без изменения кода!

Шаг 4: Создание автокликера

Новая кнопка

Создайте новую кнопку, которая будет при нажатии добавлять 1 автоклик в секунду за 10 монет.


Скрипт магазина-автокликера 
func _on_button_autoclicker_pressed() -> void:
	# Проверка наличия монет у игрока
	if coins >= 10:
		# Отнимаем монеты у игрока
		coins -= 10
		# Добавляем монеты в секунду в автокликер
		coins_per_second += 1

Лайфхаки и профессиональные советы

1. Создание глобальных переменных

Создайте скрипт (Project → Project Settings → Autoload) для глобальных переменных. Это упростит доступ к данным из любой сцены.

2. Создайте префабы (заготовки)

Сделайте элементы префабами (или заготовками). Это позволит переиспользовать их и легко добавлять новые.

3. Балансируйте игру

Используйте такую формулу для стоимости улучшений: базовая_цена × множитель^количество. Начните с множителя 1.15 и регулируйте по ощущениям.

4. Тестируйте на мобильных

Если планируете выпустить игру на мобильные устройства, тестируйте управление касанием и размеры UI на разных экранах.

5. Добавьте достижения

Создайте систему достижений (например, "100 кликов", "Купить 10 улучшений"). Это увеличивает вовлечённость игроков.

6. Оптимизируйте

Используйте удаление (queue_free()) для удаления объектов и избегайте создания слишком многих нод в одной сцене.
